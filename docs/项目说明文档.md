# 求职助手（Job Application Assistant）
## 腾讯校园 AI 项目大赛 · 参赛说明文档

---

## 一、项目背景与痛点

每逢求职季，大学生面临的挑战绝不只是"写好简历"——

- **信息散落**：Boss直聘、实习僧、猎聘……不同平台需要分别刷新、反复搜索
- **筛选低效**：每天面对数十条岗位，手动判断是否匹配，耗时且主观
- **跟进失控**：投出去的简历散落在各个平台，进入哪个阶段了？什么时候要跟进？靠备忘录根本管不住
- **打招呼千篇一律**：同样的自我介绍复制粘贴，HR 一眼识破

**求职助手**正是针对上述四大痛点而构建的 AI 驱动求职全流程管理平台。

---

## 二、核心功能概览

### 2.1 AI 驱动的岗位搜索与分析

#### 意向解析 → 关键词自动生成
用户只需用自然语言描述求职方向（如"数据运营/策略运营/数据分析"），系统调用大语言模型将其拆解为 6-10 个平台搜索关键词，涵盖同义表达、常见变体（如"运营分析实习""数据运营助理"等），无需手动逐一搜索。

#### 多平台 × 多城市 × 多关键词全量抓取
- **平台**：同时覆盖 Boss直聘、实习僧两大主流校招平台
- **城市**：支持一键选择多个城市（北京/上海/深圳/广州/杭州等），取并集
- **去重**：以 `(平台, 岗位ID)` 为键自动去重，跨关键词、跨城市不重复计入

#### AI 岗位匹配分析
每条岗位经大模型与用户简历逐一比对，输出：

| 字段 | 说明 |
|------|------|
| **匹配分（0-100）** | 量化匹配程度，颜色编码（绿80+/橙60+/红60-） |
| **匹配理由** | 一句话解释分数来由 |
| **匹配亮点** | 2-3条与岗位高度吻合的简历要素 |
| **潜在顾虑** | 1-2条可能的短板，提前准备回答 |

#### Boss 活跃度过滤
搜索时可勾选"仅今日活跃HR"，系统自动过滤非活跃 HR 发布的岗位，提高打招呼的回复率。

---

### 2.2 结构化进度管理看板

参考 Linear/Jira 敏捷看板设计，将求职流程标准化为 7 个阶段：

```
待投递 → 已投递 → 简历筛选中 → 笔试/测评 → 面试中 → 🎉 Offer → 未通过
```

**看板核心功能：**
- **拖拽流转**：直接拖动卡片到目标列，状态即时更新
- **时间线记录**：每个岗位可手动追加进展备注（如"收到笔试邀请""HR已读未回"），带时间戳自动归档
- **公司规模标签**：大厂（金色）/ 中厂（蓝色）/ 小厂（灰色），覆盖字节跳动、腾讯、阿里等 100+ 公司关键词，一键识别
- **实时漏斗统计**：页面顶部始终显示各阶段数量，宏观掌握求职进展
- **列表 / 看板双视图**：按习惯切换，支持多字段排序（分数/公司/状态/地点）

---

### 2.3 AI 打招呼 & 自动投递

#### 两种打招呼模式（用户自选）
| 模式 | 说明 |
|------|------|
| **固定文案** | 用户预设模板（默认已内置个性化自我介绍），每次发送前可编辑 |
| **AI 个性化生成** | 大模型结合简历摘要 + 岗位要求实时生成，60字内，突出核心优势 |

**发送前必过预览弹窗**：无论哪种模式，内容均可在发送前修改，杜绝发错。

#### 自动投递能力
- **Boss直聘**：自动点击"立即沟通"并发送打招呼文案
- **实习僧**：自动点击"投递简历"完成投递
- **批量投递**：一键对选中的多个岗位同时投递

---

### 2.4 演示模式与真实模式分离

系统内置 30 条高质量仿真岗位数据（涵盖大厂/中厂/小厂，多角色多城市），**无需登录任何平台**即可体验全部功能流程，适合演示和评估。

演示数据与真实数据**完全隔离**存储和展示，切换 Tab 即可互不干扰地分别管理。

## 三、技术架构

```
┌─────────────────────────────────────────────────────┐
│                    前端（SPA）                        │
│  HTML5 + Vanilla JS + CSS                            │
│  看板视图 / 列表视图 / 模态弹窗 / 实时 Toast          │
└────────────────────┬────────────────────────────────┘
                     │  REST API (JSON)
┌────────────────────▼────────────────────────────────┐
│                后端（FastAPI + Python）               │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────┐ │
│  │  搜索引擎     │  │  AI 分析引擎  │  │  投递引擎  │ │
│  │  多平台爬虫   │  │  OpenRouter  │  │  Playwright│ │
│  │  多关键词去重 │  │  LLM 匹配    │  │  反检测    │ │
│  └──────────────┘  └──────────────┘  └───────────┘ │
└────────────────────┬────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────┐
│              数据层（SQLite）                         │
│  jobs / settings / 简历文本 / Cookie 会话             │
└─────────────────────────────────────────────────────┘
```

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 后端框架 | FastAPI | 异步高性能，支持 BackgroundTasks |
| 浏览器自动化 | Patchright（Playwright 分支） | 绕过 CDP 检测，模拟人类操作 |
| AI 接口 | OpenRouter（DeepSeek V3 等） | 支持换模型，默认使用 DeepSeek V3 |
| 数据库 | SQLite + aiosqlite | 零部署，本地运行 |
| 前端 | 纯原生 JS | 无框架依赖，轻量快速 |

## 四、创新亮点

### 亮点 1：从"意向"到"关键词"的语义理解
传统求职需要用户自己想搜索词，本系统通过 LLM 将自然语言意向扩展为完整关键词集合，减少因搜词不全导致的漏检。

### 亮点 2：简历-岗位向量化比对
不只是关键词匹配，AI 理解简历的项目经历、技能深度，与岗位要求进行语义层面的对比，给出有据可查的匹配理由。

### 亮点 3：反爬虫与会话保活
- 使用 Patchright 对 Chromium 进行二进制层面的反检测修补
- 通过随机 UA、随机视口、人类化鼠标轨迹、随机延迟模拟真实用户
- Boss直聘 会话绑定浏览器指纹，系统在整个生命周期内复用同一 Context，不重新创建

### 亮点 4：全程结构化，减少认知负担
从搜索 → 分析 → 决策 → 投递 → 跟进，每一步都有系统承接，用户只需做判断，不需要维护任何手工表格。

## 五、用户价值量化（实测估算）

| 传统方式 | 使用本系统 | 节省 |
|---------|-----------|------|
| 手动在3个平台分别搜索，约30分钟/天 | 一键全平台搜索，约2分钟 | **~28分钟/天** |
| 手动判断每条岗位是否匹配，约1分钟/条×50条 | AI自动评分排序，扫一眼 | **~45分钟/次** |
| 手动撰写每条打招呼，约5分钟/条 | 模板一键/AI生成+预览，约30秒 | **~4.5分钟/条** |
| 用备忘录/Excel跟进状态，容易遗漏 | 看板可视化，拖拽即更新 | **认知负担大幅降低** |

## 六、运行方式

```bash
# 安装依赖
pip install -r requirements.txt
python -m patchright install chromium

# 启动服务
uvicorn main:app --host 0.0.0.0 --port 8000

# 浏览器访问
open http://localhost:8000
```

**系统要求**：Python 3.11+，macOS / Linux，本地运行无需服务器。
